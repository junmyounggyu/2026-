# 실차 기반 자율주행 인식 및 추적 실습

성균관대학교 AMLAB (Automotive Mechatronics Lab) 주관, 2025 겨울학기 비교과 프로그램 **"실차 기반 자율주행 인식 및 추적 실습"** 수강 기록입니다. KITTI 데이터셋을 기반으로 LiDAR·카메라 센서 데이터를 직접 다루며, 지면 제거·클러스터링·딥러닝 객체 인식·좌표계 변환·센서 퓨전·칼만 필터 기반 객체 추적까지 자율주행 인식 파이프라인 전 과정을 파이썬으로 구현했습니다.

각 실습은  제공된 뼈대 코드(TODO)에 핵심 알고리즘을 직접 채워 넣는 방식으로 진행되었으며, 이 저장소에는 실습 내용 정리와 함께 직접접 구현·개선한 코드를 담았습니다.

## 프로젝트 목표

카메라 왜곡 보정부터 LiDAR 지면 제거, 객체 클러스터링/딥러닝 인식, LiDAR-카메라 좌표계 변환, 센서 퓨전, 칼만 필터 기반 다중 객체 추적까지, 실제 KITTI 주행 데이터를 사용해 자율주행 인식 시스템의 전체 흐름을 처음부터 구현하는 것을 목표로 했습니다.

## 실습별 상세 내용

### 1. LiDAR 객체 인식 (RANSAC + DBSCAN + Complex-YOLO)

**구현한 부분**
- RANSAC 알고리즘의 반복(iteration) 루프: 3점 무작위 샘플링 → 평면 방정식 계산 → inlier/outlier 분류 → 최대 inlier 파라미터 저장 로직을 직접 구현
- 3점으로부터 평면 방정식(ax+by+cz+d=0)의 법선벡터를 벡터 외적으로 구하는 `plane_fitting` 함수 구현
- 점과 평면 사이 거리 기반으로 지면(inlier)/비지면(outlier)을 분류하는 `get_inlier_indices` 함수 구현
- 반경 기반 이웃 탐색(`find_neighbor_indices`)과 코어/노이즈 포인트 판별, 큐를 이용한 라벨 확장까지 DBSCAN 전체 로직을 직접 구현
- Complex-YOLO 추론 결과(픽셀/정규화 좌표계)를 실제 라이다 좌표계(m 단위)로 역변환하고, 복소수(im, re) 기반 yaw 각도를 복원하는 `inferenceComplexYOLO` 함수 완성
- 인식된 car/pedestrian 객체의 3D Bounding Box를 Open3D로 시각화하는 코드 작성

**직접 개선한 부분**
- RANSAC: 지면일 가능성이 높은 z축 높이 구간만 샘플링하도록 제한해 정확도 향상
- RANSAC: 피팅된 평면의 법선벡터가 z축과 크게 다르면 결과를 채택하지 않도록 검증 로직 추가 (건물/대형 차량에 둘러싸인 상황에서 지면 오검출 방지)
- RANSAC: 선택된 inlier에 대해 최소자승법으로 평면 파라미터를 재최적화해 정확도 향상
- RANSAC: inlier 비율이 목표치를 넘으면 조기 종료하도록 하여 연산 속도 개선
- DBSCAN: 전수 거리 계산 대신 kd-tree 기반 이웃 탐색으로 속도 개선
- DBSCAN: 지나치게 크거나 넓게 퍼진(건물, 벽 등) 클러스터를 후처리로 제외
- DBSCAN: 인접한 클러스터 간 재군집화 로직 추가

### 2. 왜곡 보정 및 카메라 객체 인식

**구현한 부분**
- 카메라 내부 파라미터(K)와 왜곡 계수(D, 방사/접선 왜곡)를 이용해 픽셀 단위로 왜곡을 보정하는 로직을 직접 구현 후 `cv2.undistort`와 결과 비교 검증
- Ultralytics YOLO 모델의 추론 결과(Boxes 객체)를 confidence threshold로 필터링하고, 원하는 클래스(car, person)만 추출하는 `inferenceYOLO` 함수 구현
- 클래스 이름 매핑(person → pedestrian)과 중심점 기반 좌표(x, y, w, h)를 코너 좌표(x1, y1, x2, y2)로 변환하는 좌표 변환 로직 구현
- OpenCV로 검출된 객체에 바운딩 박스와 클래스/confidence 라벨을 시각화하는 코드 작성

### 3. 좌표계 변환 (LiDAR → 카메라 → 이미지)

**구현한 부분**
- 3D LiDAR 포인트를 동차좌표계(homogeneous coordinate)로 변환
- 캘리브레이션 파라미터를 이용해 LiDAR 좌표계 → 카메라 좌표계 → 이미지 픽셀 좌표계로 순차 투영하는 변환 파이프라인 구현
- 카메라 FOV(화각) 밖에 위치한 포인트를 마스킹해 제거하는 필터링 로직 구현
- 투영된 2D 픽셀 좌표를 이용해 이미지에서 해당 색상(BGR)을 추출, LiDAR 포인트를 RGB로 컬러화하여 Open3D로 시각화하는 기능 구현 (이미지 인덱싱 시 [row, col] ↔ [y, x] 관계와 BGR 순서를 유의해서 처리)

### 4. Open3D 및 Point Cloud 처리

**구현한 부분**
- KITTI `.bin` LiDAR 데이터를 로드하고 Open3D PointCloud 객체로 시각화
- 복셀 다운샘플링(`voxel_down_sample`)을 적용해 원본/다운샘플링 결과를 동시 창으로 비교
- PCA(주성분 분석)를 통해 포인트 클라우드를 감싸는 3D Oriented Bounding Box의 중심, 회전행렬, extent(l, w, h)를 직접 계산하고, `LineSet`으로 8개 코너와 12개 모서리를 연결해 박스를 시각화

### 5. LiDAR-Camera 센서 퓨전

**정리한 내용**
- 센서 퓨전을 융합 시점 기준 Early / Mid / Late Fusion으로 구조화하여 이해
  - Late Fusion: 각 센서가 독립적으로 인식 후 2D ROI 매칭으로 결과 통합 (구조 단순, 특징 손실)
  - Early Fusion: Point Painting 등으로 원본 데이터를 초기에 결합 (정확도 이론상 최상, 캘리브레이션/동기화에 민감)
  - Mid Fusion: 각 센서의 특징맵을 BEV 등 공통 공간에서 결합 (정확도-유연성 균형, 최신 자율주행 주류 방식)

### 6. 칼만 필터 및 객체 추적 (MOT)

**구현한 부분**
- 1차원 등속도 모델 칼만 필터(`SimpleKalmanFilter`) 구현: 상태전이행렬 A, 관측행렬 H 설계, `predict_step`(상태/공분산 예측), `update_step`(칼만 이득 계산 및 상태/공분산 갱신)을 표준 칼만 필터 수식에 따라 직접 구현
- 사인 곡선을 따라 움직이는 가상의 1D 객체에 노이즈가 섞인 관측값을 주입해 필터의 추정 성능을 시뮬레이션으로 검증
- 2D 자율주행 상황(x, y, heading)으로 칼만 필터를 확장하고, heading 값과 잔차(residual)의 각도 범위를 wrap-around 처리(`angle_mod`, `residual_mod`)하는 이유를 이해하고 구현
- 프레임별 추적 파이프라인 전체 구현:
  1. 기존 추적기(tracker) 전체에 대해 predict 수행
  2. 신규 인식 결과와 기존 추적기 간 거리 행렬 계산
  3. 헝가리안 알고리즘(`linear_sum_assignment`)으로 최적 대응 관계 계산
  4. 매칭 거리가 임계값(DIST_MATCH)을 넘으면 매칭 실패로 간주해 신규 추적기 생성
  5. 매칭된 추적기는 update, 매칭 실패한 인식 결과는 신규 추적기로 등록
  6. 일정 프레임 이상 미인식된 추적기는 제거(N_REMOVE), 충분히 안정적으로 추적된 결과만 최종 결과에 기록(N_ACTIVE)
- 추적 결과를 KITTI tracking 평가 포맷으로 변환/저장하고 TrackEval(`run_kitti.py`)로 정량 평가 수행

**한계 및 개선 방향 (분석)**
- 빠르게 움직이는 객체는 초기 속도 추정이 부정확해 프레임 간 대응관계가 잘못 구해질 수 있음 → 레이더 등 속도 측정 센서와 결합해 초기 속도 추정 정확도를 높이는 방향으로 개선 가능
- 다수 객체 동시 인식 시 헝가리안 매칭이 잘못될 가능성 증가 → 카메라 기반 re-identification 등과 결합해 객체 간 구분 정확도를 높이는 방향으로 개선 가능

## 사용 기술 스택

- **언어/환경**: Python
- **3D 데이터 처리**: Open3D
- **딥러닝 객체 인식**: Complex-YOLO (LiDAR BEV 기반 3D 인식), Ultralytics YOLO (2D 이미지 인식)
- **영상 처리**: OpenCV
- **수치 연산**: NumPy, SciPy(`linear_sum_assignment`)
- **데이터셋**: KITTI (LiDAR point cloud, 카메라 이미지, 캘리브레이션, tracking 라벨)
- **평가**: TrackEval (KITTI MOT 벤치마크)

## 배운 점

RANSAC과 DBSCAN 같은 고전적인 포인트 클라우드 처리 알고리즘부터 Complex-YOLO 기반 딥러닝 3D 인식, 센서 간 좌표계 변환, 그리고 칼만 필터·헝가리안 알고리즘을 결합한 다중 객체 추적까지, 자율주행 인식 파이프라인을 하나씩 직접 구현하며 각 알고리즘의 동작 원리와 실제 주행 데이터에서 발생하는 예외 상황(오검출, 오매칭, 좌표계 오차 등)에 대한 대응 방법을 익혔습니다. 특히 알고리즘의 기본 구현에 그치지 않고 정확도·속도 측면의 개선점을 직접 찾아 적용해보며 실제 자율주행 시스템 설계에서 고려해야 할 트레이드오프를 경험했습니다.
